Getting Started

Installation

Install DataForge from PyPI using pip:

# Install the latest stable release
pip install dataforge

# Or install with optional performance extras
pip install dataforge[performance]

Verify your installation:

python -c "import dataforge; print(dataforge.__version__)"
# Output: 3.2.1

Your First Pipeline

DataForge uses a pipeline-based architecture. Here is a complete example that reads a CSV file, filters rows, transforms a column, and writes the result:

from dataforge import Pipeline, readers, transforms, writers

# 1. Create a pipeline
pipeline = Pipeline("my_first_pipeline")

# 2. Define the data source
pipeline.read(readers.csv("sales_data.csv"))

# 3. Add processing steps
pipeline.filter("amount", lambda x: x > 100)
pipeline.transform("date", transforms.to_datetime("%Y-%m-%d"))
pipeline.add_column("tax", lambda row: row["amount"] * 0.21)

# 4. Write output
pipeline.write(writers.parquet("output/sales_cleaned.parquet"))

# 5. Execute
result = pipeline.run()
print(f"Processed {result.row_count} rows in {result.elapsed_ms}ms")
Tip Pipelines are lazy by default. No data is processed until you call .run(). This allows DataForge to optimize the execution plan before processing begins.

API Reference

Pipeline(name, config=None)

Creates a new data processing pipeline.

ParameterTypeDefaultDescription
namestr—Unique identifier for the pipeline
configForgeConfig | NoneNoneOptional configuration object. Uses defaults if not provided.

Returns: Pipeline instance

pipeline = Pipeline("etl_daily", config=ForgeConfig(workers=4))

Pipeline.filter(column, predicate)

Adds a filter step that retains only rows where predicate(value) returns True.

ParameterTypeDefaultDescription
columnstr—Column name to apply the filter on
predicateCallable[[Any], bool]—Function that returns True for rows to keep

Returns: Pipeline (for chaining)

pipeline.filter("status", lambda s: s == "active")
pipeline.filter("age", lambda a: 18 <= a <= 65)

Pipeline.transform(column, func)

Applies a transformation function to every value in the specified column.

ParameterTypeDefaultDescription
columnstr—Column name to transform
funcCallable[[Any], Any]—Transformation function applied to each value

Returns: Pipeline (for chaining)

# Normalize email addresses to lowercase
pipeline.transform("email", lambda e: e.strip().lower())

Pipeline.run(dry_run=False)

Executes the pipeline and returns a result summary.

ParameterTypeDefaultDescription
dry_runboolFalseIf True, validates the pipeline without processing data

Returns: PipelineResult with attributes row_count, elapsed_ms, errors, and warnings

result = pipeline.run()
print(result.row_count)   # 48,312
print(result.elapsed_ms)  # 127

Configuration

Create a ForgeConfig object to customize pipeline behavior:

from dataforge import ForgeConfig

config = ForgeConfig(
    workers=4,
    chunk_size=10_000,
    log_level="INFO",
    error_handling="skip",
)
OptionTypeDefaultDescription
workersint1Number of parallel worker threads for processing
chunk_sizeint5000Number of rows processed per chunk
log_levelstr"WARNING"Logging verbosity: DEBUG, INFO, WARNING, ERROR
error_handlingstr"raise""raise" to stop on errors, "skip" to skip bad rows, "log" to log and continue
max_memory_mbint512Maximum memory usage in MB before spilling to disk
output_compressionstr | NoneNoneCompression for output files: "gzip", "snappy", "zstd", or None

Troubleshooting

PipelineError: Column 'X' not found

This occurs when a .filter() or .transform() references a column that does not exist in the data source. Check that the column name matches exactly (names are case-sensitive).

# Wrong: column name has a trailing space in the CSV header
pipeline.filter("amount", lambda x: x > 0)

# Fix: strip whitespace from headers
pipeline.read(readers.csv("data.csv", strip_headers=True))

MemoryError: Exceeded max_memory_mb

The pipeline exceeded the configured memory limit. Solutions:

  1. Increase max_memory_mb in your config
  2. Reduce chunk_size to process fewer rows at a time
  3. Enable output compression to reduce write buffer size

ImportError: No module named 'dataforge'

Ensure you have installed DataForge in the correct Python environment:

# Check which Python is active
which python
pip install dataforge

Pipeline runs slowly with large JSON files

JSON parsing is inherently slower than CSV. For large datasets, convert to Parquet first:

# Convert JSON to Parquet for faster subsequent reads
Pipeline("convert").read(readers.json("big.json")).write(writers.parquet("big.parquet")).run()

Migration Guide: v2 to v3

DataForge v3 introduces a new pipeline API. Here are the key changes:

1. Pipeline creation

# v2 (deprecated)
from dataforge import Forge
f = Forge()
f.load("data.csv")

# v3 (current)
from dataforge import Pipeline, readers
p = Pipeline("my_pipeline")
p.read(readers.csv("data.csv"))

2. Configuration

# v2: dict-based config
f = Forge(config={"threads": 4})

# v3: typed ForgeConfig object
from dataforge import ForgeConfig
p = Pipeline("my_pipeline", config=ForgeConfig(workers=4))

3. Removed APIs

v2 APIv3 Replacement
Forge.load()Pipeline.read()
Forge.save()Pipeline.write()
Forge.process()Pipeline.run()
config={"threads": N}ForgeConfig(workers=N)
Breaking change The Forge class was removed entirely in v3.0. All code must migrate to the Pipeline API. A compatibility shim is available via pip install dataforge-compat but will be discontinued in v3.5.
